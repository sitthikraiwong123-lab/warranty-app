# คู่มือ Setup (ล่าสุด) — Machine/Part Lookup Database

ใช้ Tab ที่มีอยู่แล้วของคุณตรงๆ ไม่ต้องเปลี่ยนชื่อ Tab หรือคอลัมน์เดิมเลย และ**ไม่มีขั้น Pending/Approve อีกแล้ว** — เครื่อง/อะไหล่ใหม่ที่พิมพ์เข้ามาจะถูกบันทึกตรงเข้าตารางหลักทันที ถ้าผิดก็ลบได้เองในชีต **หรือลบผ่านปุ่ม "📋 ดู DB" ในแอป** (ดูส่วน "เพิ่งบันทึกใหม่" ด้านบนของหน้าต่างนั้น)

## ภาพรวมการแมปข้อมูล

| ระบบ | Tab จริงของคุณ | คอลัมน์ที่แปลงชื่อให้ |
|---|---|---|
| Machines | `warranty part` | Machine Serial Number → MachineNumber, Customer name → CustomerName, Machine Type → MachineType |
| Parts | `Part No.` (มีจุด) | Article No. → ArticleNo (Description ตรงชื่ออยู่แล้ว) |

คอลัมน์อื่นที่มีอยู่ (CNC version, Warranty Over, CNC Ver, Suggestion word1-9) ไม่ถูกแก้ไขหรือลบ ใช้เป็น alias ค้นหาได้แล้วในแอป — ถ้า Suggestion word 1-9 เต็มหมด ระบบจะสร้าง word10, word11, ... ให้อัตโนมัติ ไม่ต้องสร้างมือ

## ขั้นตอน (อัปเดต Code.gs เดิม ไม่ใช่สร้างใหม่)

**รอบนี้มี Code.gs เวอร์ชันใหม่ (เพิ่มฟังก์ชันลบ) ต้องอัปเดตอีกครั้ง:**

1. เปิด Spreadsheet เดิม → **Extensions → Apps Script**
2. ลบโค้ดทั้งหมดใน `Code.gs` ออก แล้ว copy เนื้อหาจากไฟล์ `Code.gs` ใหม่ (อันนี้) มาวางทับ
3. กด **Save** (Ctrl+S)
4. **สำคัญ:** อัปเดต deployment เดิม ไม่ใช่สร้างใหม่ (เพื่อให้ URL เดิมที่ใส่ไว้ใน `index-2.html` ยังใช้ได้):
   - ไปที่ **Deploy → Manage deployments**
   - กดไอคอนดินสอ (✏️) ที่ deployment ที่ใช้งานอยู่
   - ช่อง **Version** เลือก **"New version"**
   - กด **Deploy**

ไม่ต้องแก้อะไรในไฟล์ `index-2.html` อีก เพราะ URL เดิมยังใช้ได้เหมือนเดิม

## ทดสอบ

1. เปิดแอป → กดปุ่ม "⟳ ซิงค์" ที่แถบบน
2. ดูแถบสถานะใต้ช่อง Customer ควรขึ้น "อัปเดตล่าสุด ... · เครื่อง 32 / อะไหล่ N"
3. พิมพ์ Machine Serial Number ที่มีอยู่ (เช่น `18905`) แล้วแตะออกจากช่อง → Machine Type + Customer ขึ้นเองเป็น `EXY-6-2528-160` / `Olympic`
4. พิมพ์ Article No. ที่มีอยู่ใน Part No → แตะออกจากช่อง → Description ขึ้นเอง
5. พิมพ์เลขที่ไม่มี (เครื่องหรืออะไหล่) แล้วแตะออกจากช่อง → เปิด `warranty part` หรือ `Part No` ดู จะเห็นแถวใหม่เพิ่มเข้ามาทันที (คอลัมน์อื่นที่ไม่มีข้อมูลจะเว้นว่างไว้ให้กรอกเพิ่มเอง)

## ถ้าพิมพ์ผิด/ไม่ต้องการแถวที่เพิ่งเพิ่ม

ลบแถวนั้นในชีตได้ตามปกติเลย (คลิกขวา → Delete row) ไม่ต้องผ่านขั้นตอนอนุมัติใดๆ

ถ้าอยากดูว่ามีอะไรถูกเพิ่ม/แก้ไปเมื่อไหร่ ใช้ **File → Version history → See version history** ของ Google Sheets เองได้เลย (มีอยู่แล้วในตัว ไม่ต้องเขียนโค้ดเพิ่ม)

## เพิ่มรูปภาพให้อะไหล่ (ถ้าต้องการ)

1. **เพิ่มคอลัมน์ `ImageURL`** ใน Tab `Part No.` — เพิ่มต่อท้ายหลัง Suggestion word9 ได้เลย ใส่ลิงก์รูปจาก Google Drive (รูปแบบ `https://lh3.googleusercontent.com/d/FILE_ID` หรือลิงก์ Drive สาธารณะอื่น)
2. **สร้างโฟลเดอร์ Drive** สำหรับเก็บรูปที่ถ่ายผ่านแอป → copy ID จาก URL (ส่วนหลัง `/folders/`) → ใส่ใน `Code.gs` ช่อง `IMAGE_FOLDER_ID` แล้ว Deploy New Version อีกรั้ง
3. ถ้าไม่ต้องการฟีเจอร์รูปภาพ ข้ามขั้นตอนนี้ไปได้เลย แอปยังทำงานปกติโดยไม่มีรูป

## ชุดเบิก / พาร์ทแนะนำให้เบิกเป็นชุด (Companion sets — จัดการในแอป)

ฟีเจอร์นี้: เมื่อเลือก/พิมพ์ Article No ของ **พาร์ทหลัก** ของชุด แอปจะถามว่าต้องการเบิกพาร์ทที่เหลือในชุดด้วยไหม ถ้ากด "ใช่" จะเพิ่มพาร์ทเหล่านั้นเป็นแถวใหม่ (ล็อกแก้ไข) โดยใช้ Machine No/Type เดียวกัน และ **ใส่รูปรวมของชุดให้พาร์ทหลักอัตโนมัติ**

**จัดการชุดในแอปได้เลย** — ไม่ต้องแก้ชีตมือ: "📋 ดู DB" → แท็บ **ชุดเบิก** → "＋ เพิ่มชุดเบิก" → ตั้งชื่อชุด → ค้นหาพาร์ทจาก DB แล้วกด **Add** (พาร์ทแรก = ⭐ หลัก/ตัวกระตุ้น กดปุ่ม ⭐ เพื่อเปลี่ยนตัวหลักได้) → อัปโหลด **รูปรวมพาร์ท** 1 รูป → บันทึก (แก้/ลบได้ภายหลัง)

**โครงสร้าง Tab `Companion`** (แอปเขียนให้เอง ไม่ต้องแก้มือ — แต่ถ้าเปิดดู): แต่ละแถว = 1 ชุด — คอลัมน์ A = ชื่อชุด, B = URL รูปรวม, C เป็นต้นไป = Article No ของพาร์ท (ตัวแรก = หลัก)

| A (ชื่อชุด) | B (รูปรวม) | C (หลัก) | D | E |
|---|---|---|---|---|
| WATERFILTER POLYMER | https://.../img | 72656 | 72657 | |

> ⚠️ **โครงสร้างเปลี่ยนจากเวอร์ชันก่อน** (เดิมคอลัมน์ A = พาร์ทหลักเลย) ถ้าเคยสร้าง Companion แบบเก่าไว้ ให้ลบแล้วสร้างใหม่ผ่านแอป หรือย้ายข้อมูลให้ตรงรูปแบบใหม่

ต้อง **Deploy Code.gs ใหม่** (มี `getCompanionSets`, `saveCompanionSet`, `deleteCompanionSet`) — Deploy → Manage deployments → ✏️ → New version → Deploy แล้วกด "⟳ ซิงค์". ถ้าไม่มี Tab นี้ แอปทำงานปกติ แค่ไม่มีการแนะนำชุด (ไม่ error)

## แก้ไขอะไหล่แบบมีประวัติ/กู้คืน (updatePart — ต้อง Deploy ใหม่)

ในหน้า "📋 ดู DB" → แท็บ **อะไหล่** ตอนนี้รวมรูปภาพเข้ามาแล้ว (รูปอยู่หน้ารายการ แตะเพื่อดูเต็ม) และมีปุ่ม **✎ แก้ไข** ต่อรายการ ใช้แก้ รูป / รหัส (Article No) / Description / predict word ได้

**ความปลอดภัยของข้อมูล:** ก่อนแก้ไขทุกครั้งระบบ **บังคับให้ลงชื่อผู้แก้ไข** และ **สำรองข้อมูลเดิม** ลงแท็บ `EditLog` โดยอัตโนมัติ แล้ว **กู้คืนได้** (ปุ่ม 🕘 ประวัติ / กู้คืนข้อมูล ในหน้าต่างแก้ไข)

**วิธีเปิดใช้:**
1. อัปเดต `Code.gs` เวอร์ชันใหม่ (มี `updatePart`, `getEditLog`, `restorePart`) แล้ว **Deploy → Manage deployments → ✏️ → New version → Deploy**
2. ไม่ต้องสร้างแท็บ `EditLog` เอง — ระบบสร้างให้อัตโนมัติครั้งแรกที่มีการแก้ไข (คอลัมน์: Timestamp, Editor, Action, ArticleNo, NewArticleNo, OldData, NewData)
3. **ข้อควรระวังเรื่องเปลี่ยนรหัส (Article No):** การเปลี่ยนรหัสจะเปลี่ยน "กุญแจ" ของอะไหล่ ทำให้ลิงก์รูป/alias/Companion ที่อ้างอิงรหัสเดิมอาจไม่ตรงกัน ถ้าเปลี่ยนรหัสควรตรวจ Companion sheet ด้วย ระบบจะปฏิเสธถ้ารหัสใหม่ซ้ำกับที่มีอยู่แล้ว

## ลงชื่อเข้าใช้ก่อนแก้ไข + ลบแบบกู้คืนได้ / ถังขยะ (softDeletePart — ต้อง Deploy ใหม่)

**ลงชื่อก่อนแก้ไข:** กดปุ่ม **✎ แก้ไข** ที่อะไหล่ **ครั้งแรกในแท็บนั้น** จะมี **หน้าต่างลงชื่อเข้าใช้ (🔒)** ขึ้นก่อน ต้องพิมพ์ชื่อผู้แก้ไขแล้วกด "เข้าสู่ระบบ" จึงจะเข้าหน้าแก้ไขได้ (ชื่อนี้ถูกบันทึกในประวัติ) — เป็นการลงชื่อเพื่อระบุตัวผู้แก้ไข ไม่ใช่ระบบรหัสผ่าน หลังจากนั้นแก้ไขอะไหล่ชิ้นอื่นในแท็บเดิมจะไม่ถามซ้ำ (ดูหัวข้อ "เซสชันแบบผูกกับแท็บ" ด้านล่าง) ในหน้าแก้ไขมีปุ่ม "เปลี่ยนผู้ใช้" ให้ลงชื่อใหม่ได้

**ลบแบบกู้คืนได้ (soft delete):** ในหน้าแก้ไขมีปุ่ม **🗑 ลบอะไหล่นี้ (ย้ายไปถังขยะ)** เมื่อลบ ระบบจะ **ไม่ลบถาวรทันที** แต่จะคัดลอกทั้งแถวไปเก็บที่แท็บ `RecycleBin` (พร้อมวันเวลา + ชื่อผู้ลบ) แล้วเอาออกจากตารางหลัก `Part No.` เปิดแท็บ **🗑 ถังขยะ** ใน "📋 ดู DB" เพื่อดูรายการที่ถูกลบ และกด **↩ กู้คืน** (ต้องลงชื่อเข้าใช้อีกครั้ง) เพื่อนำกลับเข้าฐานข้อมูล

**วิธีเปิดใช้:**
1. อัปเดต `Code.gs` เวอร์ชันใหม่ (มี `softDeletePart`, `getRecycleBin`, `restoreDeletedPart`) แล้ว **Deploy → Manage deployments → ✏️ → New version → Deploy**
2. ไม่ต้องสร้างแท็บ `RecycleBin` เอง — ระบบสร้างให้อัตโนมัติครั้งแรกที่มีการลบ (คอลัมน์: DeletedAt, DeletedBy, ArticleNo, Description, RowData โดย RowData เก็บทั้งแถวเดิมเป็น JSON เพื่อกู้คืนได้ครบทุกคอลัมน์)
3. การกู้คืนจะถูกปฏิเสธถ้ามีรหัสเดียวกันกลับเข้ามาในตารางหลักแล้ว (กันข้อมูลชนกัน)

**เซสชันแบบผูกกับแท็บ + ออกจากระบบอัตโนมัติเมื่อปิดหน้า (auto-logout):** เมื่อลงชื่อเข้าใช้ครั้งแรก ระบบจะจำชื่อผู้แก้ไขไว้ใน **`sessionStorage` (ผูกกับแท็บนั้น ๆ)** — ตราบใดที่แท็บ/หน้าต่าง "📋 ดู DB" ยังเปิดอยู่ การกดแก้ไขอะไหล่ชิ้นต่อ ๆ ไป **จะไม่ถามให้ลงชื่อใหม่** (รวมถึงรีเฟรชหน้าเดิมก็ยังจำได้) มีปุ่ม "เปลี่ยนผู้ใช้" ในหน้าแก้ไขไว้เปลี่ยนตัวผู้ลงชื่อได้เอง เมื่อ **ปิดแท็บ/หน้าต่าง** เบราว์เซอร์จะล้าง `sessionStorage` ให้อัตโนมัติ (ครั้งหน้าเปิดใหม่หรือเปิดอีกแท็บต้องลงชื่อใหม่) และตอนปิดหน้าแอปจะส่งสัญญาณไปบันทึก "logout" ที่แท็บ `EditLog` แบบ best-effort (`navigator.sendBeacon`) หมายเหตุ: แอปนี้ไม่มี session token ฝั่งเซิร์ฟเวอร์ — "เซสชัน" คือชื่อผู้แก้ไขที่เก็บใน `sessionStorage` ของแท็บ (การส่ง logout จะเกิดตอนรีเฟรชด้วย ซึ่งเป็นแค่ noise ในบันทึก ไม่กระทบการใช้งาน) และต้องมี action `logout` ใน `Code.gs` เวอร์ชันใหม่

## เพิ่มเครื่อง / อะไหล่ใหม่ ในหน้า “ข้อมูล Database” (ไม่ต้อง Deploy ใหม่)

หน้า “📋 ดู DB” ถูกปรับดีไซน์ให้อ่านง่ายขึ้น (แท็บแบบ segmented, ตารางสลับสีแถว, การ์ดมีเงา) และเพิ่มปุ่มเพิ่มข้อมูลใหม่ 2 จุด:

- **แท็บ “เครื่อง” → ปุ่ม “＋ เพิ่มเครื่องใหม่”**: กรอก Customer Name (พิมพ์แล้วมีรายการแนะนำจากลูกค้าที่เคยมีในฐานข้อมูล), Machine No (บังคับ), Machine Type/Model (มีรายการแนะนำจากรุ่นที่เคยมี) แล้วกดบันทึก
- **แท็บ “อะไหล่” → ปุ่ม “＋ เพิ่มอะไหล่ใหม่”**: กรอก Article No., Part Name, Predict Word (คำค้นหาเพิ่ม — ต้องมี Article No.) กด **“＋ เพิ่มบรรทัด”** เพื่อกรอกหลายอะไหล่พร้อมกันก่อนบันทึกทีเดียว

ทั้งสองใช้ action `recordNew` ที่มีอยู่แล้วใน `Code.gs` (Predict Word ถูกบันทึกเป็น Suggestion word ผ่านการเพิ่มชื่อซ้ำบน Article No. เดิม) จึง **ไม่ต้อง Deploy Code.gs ใหม่** สำหรับฟีเจอร์นี้ ข้อมูลจะแสดงทันที (optimistic) และซิงค์กับเซิร์ฟเวอร์เบื้องหลังอีกครั้ง

## รูปอะไหล่ไม่แสดง / พรีวิวไม่ขึ้น (repairImages — ต้อง Deploy ใหม่)

รูปจะแสดงใน `<img>` ได้ก็ต่อเมื่อไฟล์ใน Drive เปิดแชร์แบบ **"anyone with link"** — รูปที่อัปโหลดผ่านแอปถูกตั้งแชร์ให้อัตโนมัติ แต่ **ลิงก์ที่คัดลอกมาวางเองจากแอป Drive** (เช่น `.../view?usp=drivesdk`) มักเป็นไฟล์ส่วนตัว ทำให้รูปไม่ขึ้นตลอดไป ระบบจัดการให้เอง **โดยไม่มีปุ่มให้กด**:

1. **อัปโหลดตอนกด "บันทึก" เท่านั้น (แทนที่รูปเดิม):** การเลือกรูปในฟอร์มแก้ไข/เพิ่มอะไหล่เป็นแค่พรีวิวในเครื่อง — ไฟล์จะขึ้น Drive **ครั้งเดียว**ตอนกดบันทึก และรูปเก่าของอะไหล่ตัวนั้นถูกย้ายลงถังขยะ Drive อัตโนมัติ (แทนที่ ไม่สะสมไฟล์ซ้ำ) ถ้าอัปโหลดล้มเหลวระบบจะยังไม่บันทึกและแจ้งให้ลองใหม่
2. **ตั้งแชร์อัตโนมัติตอนบันทึก:** ทุกครั้งที่บันทึก URL รูป (`setImageURL` / `updatePart`) ระบบพยายามตั้งสิทธิ์ไฟล์เป็น anyone-with-link (ถ้าองค์กรบล็อก จะลอง domain-with-link แทน) — แต่**ไม่ล้มถ้าตั้งไม่ได้**

**สำคัญ — องค์กรที่บล็อกการแชร์สาธารณะ (เจอ `Access denied: DriveApp` ตอนอัปโหลด):** Google Workspace หลายองค์กรห้ามแชร์ไฟล์แบบ "anyone with link" ด้วยนโยบายความปลอดภัย ทำให้ไฟล์อัปโหลดขึ้น Drive ได้แต่ตั้งแชร์สาธารณะไม่ได้ (รูปเลยไม่แสดงใน `<img>` ตลอด) แก้แล้วด้วย **image proxy**: (ก) `uploadImage` ไม่ล้มถ้าแชร์ไม่ได้ (สร้างไฟล์สำเร็จ + คืน URL ปกติ) (ข) action ใหม่ `getImageBytes` ให้ web app (ซึ่งรันในสิทธิ์เจ้าของ อ่านไฟล์ได้เสมอ) ส่ง byte รูปกลับมาเป็น base64 (ค) เมื่อรูปโหลดจาก Drive ตรง ๆ ไม่ขึ้น แอปจะดึงผ่าน proxy อัตโนมัติแล้วแสดงเป็น data URL (แคชตาม file id, พอ proxy สำเร็จครั้งแรกรูปถัด ๆ ไปจะข้ามการรอ retry เลย) — รูปจึงแสดงได้แม้องค์กรจะปิดการแชร์สาธารณะ **ต้องคง web app deployment เป็น "Execute as: Me (เจ้าของ)" + "Who has access: Anyone"** เหมือนเดิม
3. **ซ่อมอัตโนมัติเบื้องหลัง (action `repairImages`):** ทำงานเองครั้งเดียวต่อการเปิดแอป เมื่อเปิดแท็บ "อะไหล่" หรือเมื่อพบรูปที่โหลดไม่ขึ้น — (ก) ไล่ตั้งสิทธิ์แชร์รูปทุกไฟล์ในคอลัมน์ ImageURL และ (ข) **อะไหล่ที่ยังไม่มีรูปแต่มีไฟล์ค้างอยู่ในโฟลเดอร์ Drive** (ชื่อไฟล์ขึ้นต้นด้วยรหัสอะไหล่ เช่น `46694_...`) จะถูกผูกรูปไฟล์ล่าสุดให้อัตโนมัติ — รูปที่เคยอัปโหลดแล้วหลุด (ไม่ได้กดบันทึก) จะกลับมาเอง
4. **กันลืมกดบันทึก:** ถ้าแนบ/เปลี่ยนรูปแล้วพยายามปิดฟอร์ม ระบบจะถามยืนยันก่อน

ต้อง **Deploy `Code.gs` เวอร์ชันใหม่** (มี `repairImages` แบบผูกรูปจากโฟลเดอร์ + `ensureImageShared_`) จึงจะได้ข้อ 2–3

## ถ้ายังมีปัญหาเรื่องบันทึก/ค้นหาไม่เจอ

โค้ดรอบนี้ทำให้ทนทานกับชื่อคอลัมน์ที่มีช่องว่างแฝง/ตัวพิมพ์เล็กใหญ่ต่างกันแล้ว แต่ถ้ายังมีปัญหาอยู่ เปิดลิงก์นี้ในเบราว์เซอร์ (เปลี่ยน URL เป็นของคุณ) เพื่อดูชื่อคอลัมน์จริงแบบเห็นช่องว่างชัดๆ (มี `"..."` ครอบไว้):

```
https://script.google.com/macros/s/YOUR_ID/exec?action=debugHeaders
```

ส่งผลลัพธ์ที่ได้มาให้ดูได้เลยครับ จะช่วยหาสาเหตุได้ตรงจุดทันที

## หมายเหตุ

- ไม่มีระบบ login/password — เหมาะกับทีมเล็กที่ไว้ใจกัน
- ไม่มีระบบ Pending/Approve แล้ว — ข้อมูลใหม่เข้าตารางหลักทันที ตามที่ตกลงกัน เพื่อความเร็ว/ความง่าย แลกกับการไม่มีขั้นตรวจสอบก่อนข้อมูลขึ้นระบบจริง
- Tab `MC List` และ `Part No.` เดิมไม่ถูกแก้ไขโครงสร้างเลย โค้ดแค่ "แปลชื่อคอลัมน์" ตอนอ่าน/เขียนเท่านั้น
- **กันข้อมูลเก่าทับการลบ:** ตอนลบผ่านปุ่ม "📋 ดู DB" ระบบจะสร้าง Tab `DeletedKeys` อัตโนมัติเก็บประวัติการลบไว้ 7 วัน กันมือถือเครื่องอื่นที่มีข้อมูลเก่าค้าง (เช่น พิมพ์ไว้ตอนออฟไลน์) ส่งย้อนหลังไปสร้างของที่ถูกลบไปแล้วซ้ำขึ้นมาใหม่
- **อะไหล่ที่ยังไม่รู้รหัส:** พิมพ์แค่ชื่อได้ ระบบรอจนกด "บันทึก"/"Export PDF" ถึงบันทึก (ไม่บันทึกทันทีตอนพิมพ์) ค้นหาเจอได้เหมือนปกติ พอรู้ Article No. ทีหลังจะเติมให้กับแถวเดิม ไม่สร้างซ้ำ และถ้าชื่อไปตรงกับอะไหล่ที่มีรหัสอยู่แล้ว จะย้ายไปเป็น alias ของรหัสนั้นแล้วลบแถวเดิมทิ้งให้อัตโนมัติ
